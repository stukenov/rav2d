## Основные блоки работы

### 1. FFI-слой к asm (~2–3 недели)
- `extern "C"` объявления для всех asm-функций
- Таблицы указателей на функции (mc_fn, ipred_fn, cdef_fn)
- CPU feature detection (cpu.c)

### 2. Типы и структуры данных (~2–3 недели)
- headers.h — enum/struct для OBU-заголовков
- internal.h — Dav2dContext, Dav2dFrameContext, Dav2dTileState
- levels.h, tables.h — константы и LUT

### 3. Битовый ввод-вывод и энтропийное кодирование (~2–3 недели)
- getbits.c — чтение битов
- msac.c — мультисимвольный арифметический кодер

### 4. OBU-парсинг (~3–4 недели)
- obu.c (2.8K) — разбор всех типов OBU

### 5. Главный декодер (~6–8 недель)
- decode.c (5.8K) — декодирование суперблоков
- recon_tmpl.c (4.3K) — реконструкция пикселей
- ipred_tmpl.c, mc_tmpl.c — предсказание
- cdef_apply_tmpl.c, db_apply_tmpl.c, lr_apply_tmpl.c — фильтры

### 6. Многопоточность (~3–4 недели)
- thread_task.c — планировщик задач

### 7. Остальное ядро (~3–4 недели)
- refmvs.c, cdf.c, lf_mask.c, scan.c, wedge.c, ctx.c, pal.c, warpmv.c
- picture.c, ref.c, data.c, mem.c

### 8. Публичный API + утилиты (~2–3 недели)

## Оценка сроков

| | Оптимистичная | Реалистичная | Пессимистичная |
|--|:---:|:---:|:---:|
| Срок | 4–5 мес | 6–8 мес | 10–12 мес |
| Команда | 2–3 чел | 2–3 чел | 2–3 чел |
| Человеко-месяцы | 10–15 | 15–20 | 25–30 |

Один опытный разработчик (Rust + видеокодеки): 8–12 месяцев.

С использованием c2rust для начальной транспиляции: 5–9 месяцев.

## Ключевые технические вызовы

1. **Шаблонирование по битовой глубине** — _tmpl.c файлы компилируются дважды с разным BITDEPTH. Решение: Rust generics по типу пикселя.
2. **unsafe** — реалистично 20–30% кода будет unsafe (FFI, DSP-таблицы, буферы пикселей).
3. **Спецификация AV2 меняется** — dav2d ещё в разработке.
4. **Бит-точность** — каждый модуль верифицируется conformance-тестами.
5. **Многопоточность** — кастомный thread pool → std::thread + Mutex/Condvar или crossbeam.

## Рекомендуемый порядок переноса

```
1. Типы/структуры + FFI-слой к asm
2. getbits + msac (энтропия)
3. obu.c (парсинг)
4. tables + cdf
5. decode.c + _tmpl.c файлы (ядро)
6. thread_task.c (многопоточность)
7. lib.c (публичный API)
8. CLI/тесты
```
